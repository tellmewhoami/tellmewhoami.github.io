Openclaw学习日志

在使用技能时我需要配置api key，以下是介绍什么是api，在不同功能中使用了哪些api key，以及怎么获得api key的途径。

什么是API

用户之前一直在配置OpenClaw的联网搜索功能，涉及Brave Search API和API Key。所以可以自然地将这两个概念融入到解释中，以用户的上下文为例。

一个简单的比喻：餐厅点餐

想象你去一家热门餐厅吃饭：

  • 你：就是你自己（或者你使用的软件应用，比如 OpenClaw）。
  • 餐厅的后厨：就是提供某种服务的服务器，它拥有强大的“能力”，比如能做各种菜肴（能执行搜索、翻译等任务）。
  • 菜单：就是 API（应用程序编程接口）。它明确列出了餐厅能提供哪些菜（有哪些功能）、每道菜叫什么名字（函数名）、需要你提供什么配料才能做（需要传入的参数），以及做好的菜会以什么形式端出来（返回的数据格式）。你通过看菜单就知道怎么点菜。
  • 服务员：就是 API 调用的过程。你把按照菜单写好的“点餐单”（API 请求）交给服务员，服务员把它送到后厨。
  • 你的会员卡或预约号：就是 API Key（API 密钥）。它证明了你的身份和权限。服务员看到你的会员卡，就知道你是本店会员，可以享受免费饮料（对应 API 的免费额度），或者你是高级会员，可以点隐藏菜单（对应更高的权限）。

🔹 什么是 API？

API 的全称是 Application Programming Interface，即“应用程序编程接口”。

简单来说，API 就是不同软件程序之间进行通信和交换数据的约定。它定义了一套规则，让你（你的程序）可以请求另一个程序的功能或数据，而不需要知道对方内部是如何实现的。

  • 它像一个“中介”或“翻译官”：它把你（你的程序）的需求，翻译成目标程序（如 Brave 的搜索服务器）能理解的指令，然后把结果再翻译回来给你。
  • 它让你能“借用”别人的能力：比如，你的 OpenClaw 本身不会搜索网页，但它可以通过调用 Brave Search API，借用 Brave 搜索引擎的能力，把搜索结果拿回来给你。

🔑 什么是 API Key？

API Key 就是 API 密钥，它是一个唯一的、加密的字符串，用来标识你的身份，并授权你使用某个 API。

当你申请使用一个 API 服务（比如 Brave Search）时，服务商会给你一个 API Key。每次你的程序（OpenClaw）调用该 API 时，都必须附上这个 Key，就像每次点餐都要出示会员卡一样。

API Key 的主要作用有：

  1. 身份验证：证明“我是我”，告诉服务器是谁在调用。
  2. 授权与控制：服务器根据你的 Key，判断你有哪些权限（比如免费用户每天只能调用 100 次，付费用户可以不限量）。
  3. 计量与计费：服务器会记录每个 Key 的调用次数，用于统计使用量，或者在超出免费额度时向你收费。
  4. 安全防护：防止 API 被恶意滥用。如果某个 Key 有异常请求，服务器可以封禁这个 Key，而不影响其他用户。

📌 结合你之前的场景

现在，我们再来看 OpenClaw 和 Brave Search API 的关系，就非常清晰了：

  • Brave 公司：提供强大的搜索服务，但它不能直接让你随便用，因为需要控制成本。
  • Brave Search API：是 Brave 公司为开发者提供的一个“官方点餐窗口”（菜单），它规定了如何正确地请求搜索功能。
  • 你的 OpenClaw 程序：是一个“顾客”，它想使用 Brave 的搜索能力。
  • 你申请的 Brave API Key：就是你从 Brave 那里拿到的“会员卡”。OpenClaw 每次向 Brave 发起搜索请求时，都会在“点餐单”里附上这张会员卡，证明“我是合法的用户，请帮我搜索”。
  • 如果不给 Key：Brave 的服务器不认识你是谁，就会拒绝提供服务（返回错误），就像餐厅不让没会员卡的人点餐一样。

所以，OpenClaw 要求你配置 Brave API Key，本质上是帮你把它自己变成了一个经过授权的合法顾客，这样才能顺利地从 Brave 那里获取搜索信息。

希望这个解释能帮你彻底弄明白这两个概念。

Multiple-Search-Engine Skill

以下是multiple-search-engine能力的使用介绍和api获取方式。s
要使用OpenClaw的联网搜索功能，你需要先获取Brave Search的API密钥，然后在OpenClaw中进行配置；最后需要enable web-fetch。整个过程分为两个步骤。

第一步：查询并获取你的 Brave API Key

  1. 访问官网并登录/注册
    打开浏览器，访问 Brave Search API 官方网站：https://brave.com/search/api/ 。如果你没有账号，需要先注册一个；如果有，直接登录。

  2. 进入API Keys管理页面
    登录成功后，在仪表盘的左侧菜单栏中找到并点击 “API Keys” 选项卡 。

  3. 创建新的API Key
    点击 “ADD API Key” 按钮 。系统可能会提示你先选择一个订阅计划（Subscription plan）。对于个人使用和测试，通常有免费的套餐可供选择，例如每月提供 2,000 次搜索请求的免费计划，这对个人用户通常足够 。

  4. 生成并保存你的Key
    为你的API Key输入一个名称（例如 “openclaw”），选择好计划后，点击创建。生成后，请务必立即复制并妥善保存你的API Key。出于安全考虑，这个密钥通常只会显示一次 。

🔍 为什么是 Brave Search API？

OpenClaw 本身只是一个“桥梁”工具，它本身并不具备搜索能力，需要调用外部的搜索引擎 API 来为你获取网络信息。选择 Brave Search API，主要有以下几个原因：

  • 注重隐私：Brave Search 主打隐私保护，不追踪用户，这与很多开源工具的价值观契合。
  • 有免费额度：Brave Search API 提供免费的套餐（例如每月前 2,000 次搜索免费），对于个人开发者和小型项目非常友好。
  • 简单易用：它的 API 设计相对简洁，集成成本低，适合像 OpenClaw 这样的工具快速接入。
  • 无广告干扰：返回的结果比较干净，没有广告，适合程序解析和处理。

当然，OpenClaw 也可能支持其他搜索引擎（比如 Google、Bing 的 API），但 Brave Search 是目前文档中明确支持的默认选择。

🔑 为什么需要你自己的 API 密钥？

因为 OpenClaw 是运行在你的电脑上的本地工具，而不是一个中心化服务。它调用 Brave Search API 时，Brave 需要知道是哪个用户在调用，以便：

  1. 计量和限制：判断你是否在免费额度内，防止滥用。
  2. 计费：如果你使用了付费套餐，需要向你的账户收费。
  3. 安全：确保调用者是合法用户，而不是恶意爬虫。

所以，你需要自己去 Brave 官网申请一个专属的 API 密钥，然后告诉 OpenClaw 使用这个密钥。这就像你要用水，需要自己办一张水卡，然后插到水表里才能出水。

⚠️ 如果 API 密钥被盗会有什么风险？

API 密钥相当于你在 Brave 的“身份凭证”。如果被盗，后果取决于你的账户设置：

  • 消耗免费额度：如果攻击者用你的密钥大量搜索，你的免费额度会被快速耗尽，导致你自己无法正常使用搜索功能。
  • 产生意外费用：如果你绑定了信用卡且开启了自动续费，或者不小心订阅了付费计划，攻击者滥用可能导致你产生意料之外的费用，甚至可能被刷爆。
  • 账户关联风险：虽然 Brave Search API 本身不涉及你的个人信息（如聊天记录），但密钥与你的 Brave 账户绑定，如果 Brave 账户还有其他敏感信息，可能会有间接风险。
  • 被用于非法用途：攻击者可能利用你的密钥进行爬虫、数据抓取等，一旦被 Brave 判定为滥用，你的整个账户可能被封禁。

因此，保护你的 API 密钥非常重要：

  • 不要将其硬编码在代码中，尤其是不要提交到公开的 GitHub 仓库。
  • 不要在论坛、聊天群等公开场合分享。
  • 如果怀疑密钥泄露，应立即登录 Brave 仪表盘重新生成（撤销）旧密钥，并更新 OpenClaw 的配置。

总之，OpenClaw 选择 Brave Search 是出于隐私和成本的考量，而你需要自己申请密钥，这是调用任何商业 API 的标准流程。妥善保管密钥就能安全使用。

这个选项 Enable web_fetch (keyless HTTP fetch)? 是在配置 OpenClaw 的联网功能时出现的另一个重要选项。它和刚才我们讨论的 web_search（需要 Brave API Key）是两种不同的网络能力。我来帮你理清它们的区别，以及你应该如何选择。

在设置完web_search后，还需要设置web_fetch，一个是链接搜索功能，一个是网页抓取功能。

🔍 web_search vs. web_fetch

功能 web_search (网络搜索) web_fetch (网络获取)
作用 搜索：根据你的关键词，从互联网上找到相关的网页链接列表（类似你在百度/Google 搜索时看到的结果页）。 获取：直接读取指定网页的具体内容（HTML 文本），比如一篇新闻文章、一个页面的正文。
需要 API Key？ 是，需要 Brave Search API Key（或其他搜索引擎的 Key）。因为这是调用商业搜索引擎的付费/限量服务。 通常不需要，它就是你的程序像浏览器一样去访问一个网址，拉取该网址的源代码。这是普通的 HTTP 请求，没有集中的服务商来收费。
典型用途 你不知道具体哪个网页有答案，需要先通过关键词找到一批相关网址。 你已经知道某个具体的网址（比如通过搜索得到的链接），想进一步读取该网页的详细内容，以便让 AI 总结或分析。
类比 用百度搜关键词，得到一堆蓝色标题链接。 点击其中某个链接，打开网页看到里面的文字、图片。

❓ 为什么 OpenClaw 要分开询问？

因为这两个功能是解耦的，你可以根据实际需要选择开启哪一个或都开启：

  • 如果你只需要在对话中让 AI 搜索资料，但不需要 AI 自己去读取网页全文，那么只开 web_search 就够了。
  • 如果你希望 AI 能够 “看” 具体的网页内容（比如你给 AI 一个网址，让它总结文章），那么你需要开启 web_fetch。
  • 两者结合使用效果最好：AI 先通过 web_search 找到相关链接，再用 web_fetch 读取链接里的内容，最后整合答案给你。

✅ 我应该启用 web_fetch 吗？

建议你启用，因为：

  1. 它不需要申请任何密钥，配置成本为零。
  2. 能极大扩展 AI 的能力：让 AI 不仅能搜到链接，还能真的“阅读”网页里的信息，回答更深入、更具体。
  3. 通常没有额外风险：它本质上就是你的电脑向目标网站发起一次普通访问，和你自己用浏览器打开网页是一样的。唯一的区别是，如果短时间内大量抓取某个网站，可能会触发该网站的反爬机制（但个人使用一般不会）。

所以，在配置向导中遇到这个问题时，直接选择“是”或“启用”即可。 如果你以后不想用，也可以随时在配置文件里关闭。

📌 小结

  • web_fetch 是 无密钥的网页抓取功能，用来读取指定网址的内容。
  • 它与需要 API Key 的 web_search 互补，建议同时开启，以获得完整的联网问答体验。
  • 启用它没有技术门槛，也不需要额外付费。

现在你应该清楚这两个选项的区别了，可以放心地进行配置了！如果在配置过程中还有其他疑问，随时再来问我。
